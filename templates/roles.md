# Agent Roles

本文件用于目标项目仓库，说明该项目中各类 Agent 的职责边界。

## Planner Agent

负责：

- 分析需求是否值得做
- 给出备选方案
- 拆分任务
- 写验收标准
- 识别风险

不负责：

- 直接改代码
- 运行测试
- 合并 PR

## Backend Agent

负责：

- 后端接口
- 数据处理逻辑
- 数据库访问
- 后端单元测试

约束：

- 不要修改前端文件
- 不要修改部署配置，除非任务明确要求
- 不要提交密钥或 `.env`

## Frontend Agent

负责：

- 页面组件
- 交互逻辑
- 前端状态管理
- 前端测试

约束：

- 不要修改后端核心业务逻辑
- API 字段变更必须同步说明

## Tester Agent

负责：

- 测试计划
- 单元测试
- 接口测试
- 回归测试
- 分析测试失败日志

约束：

- 测试命令应在受控环境中运行
- 不要修改生产数据

## Reviewer Agent

负责：

- 审查方案和代码
- 检查风险
- 检查验收标准是否满足
- 判断是否需要返工

约束：

- 不直接改代码
- 不无脑通过

## Docs Agent

负责：

- README
- 使用说明
- API 文档
- 部署说明
- 变更记录

## DevOps Agent

负责：

- Dockerfile
- docker-compose.yml
- CI/CD
- 部署脚本

高风险操作必须用户确认：

- 部署线上环境
- 修改数据库结构
- 删除远程资源
- 修改权限配置
